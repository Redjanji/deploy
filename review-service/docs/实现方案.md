以下为审核服务 (review-service) 的完整实现方案，基于你的微服务架构，与 property-service 通过 RabbitMQ 异步解耦，支持机审和人工审核。

------

## 一、服务定位与职责

- **功能**：接收房源变更事件，对房源图片和文本进行内容安全审核（机审 + 人工审核），审核结果回调 property-service 更新房源状态。
- **通信方式**：
  - 通过 RabbitMQ 消费 `property.audit` 队列中的审核事件
  - 通过 REST API 回调 property-service 的内部接口更新审核结果
- **数据库**：`review_db`，记录审核任务和结果
- **端口**：`8091`
- **依赖**：Nacos（注册/配置）、RabbitMQ、MyBatis-Plus、RestTemplate、阿里云/腾讯云内容安全 SDK（可选）

## 三、数据库设计

sql

```
CREATE DATABASE IF NOT EXISTS review_db DEFAULT CHARACTER SET utf8mb4;

USE review_db;

-- 审核任务表
CREATE TABLE audit_tasks (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    property_id BIGINT NOT NULL COMMENT '房源ID',
    app_id VARCHAR(32) COMMENT '应用ID',
    task_type VARCHAR(20) DEFAULT 'MACHINE' COMMENT 'MACHINE / MANUAL',
    status TINYINT DEFAULT 0 COMMENT '0待处理 1处理中 2通过 3拒绝 4异常',
    result_detail TEXT COMMENT '审核结果详情JSON',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_property_id (property_id),
    INDEX idx_status (status)
) COMMENT='审核任务';

-- 审核记录（机审结果、人审工单）
CREATE TABLE audit_records (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    task_id BIGINT NOT NULL,
    audit_type VARCHAR(20) COMMENT 'MACHINE / MANUAL',
    result TINYINT COMMENT '1通过 2拒绝',
    reason VARCHAR(500),
    auditor_id BIGINT COMMENT '审核员ID（人审时）',
    audit_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (task_id) REFERENCES audit_tasks(id)
) COMMENT='审核记录';
```

    public static final String QUEUE = "property.audit.queue";

### 2. AuditService 接口

public interface AuditService {

    void manualReview(Long taskId, boolean approved, String reason, Long auditorId);
}
```



### 3. AuditServiceImpl 实现

java

```
@Service
@RequiredArgsConstructor
@Transactional
@Slf4j
public class AuditServiceImpl implements AuditService {
    private final AuditTaskMapper taskMapper;
    private final AuditRecordMapper recordMapper;
    private final MachineAuditService machineAuditService;
    private final PropertyServiceClient propertyClient;

    @Override
    public void process(PropertyAuditEvent event) {
        // 1. 创建任务
        AuditTask task = new AuditTask();
        task.setPropertyId(event.getPropertyId());
        task.setAppId(event.getAppId());
        task.setTaskType("MACHINE");
        task.setStatus(0); // 待处理
        taskMapper.insert(task);

        // 2. 机审
        AuditRecord record = new AuditRecord();
        record.setTaskId(task.getId());
        record.setAuditType("MACHINE");
        try {
            boolean passed = machineAuditService.audit(event);
            record.setResult(passed ? 1 : 2);
            record.setReason(passed ? null : "内容违规");
            recordMapper.insert(record);

            // 更新任务状态
            task.setStatus(passed ? 2 : 3);
            task.setResultDetail(passed ? "机审通过" : "机审拒绝");
            taskMapper.updateById(task);

            // 3. 通知 property-service
            propertyClient.notifyAuditResult(event.getPropertyId(), passed);
        } catch (Exception e) {
            log.error("机审异常 propertyId={}", event.getPropertyId(), e);
            task.setStatus(4); // 异常
            taskMapper.updateById(task);
        }
    }

    @Override
    public void manualReview(Long taskId, boolean approved, String reason, Long auditorId) {
        AuditTask task = taskMapper.selectById(taskId);
        if (task == null || !task.getStatus().equals(3)) { // 只有机审拒绝的才需要人审
            throw new BusinessException("任务状态不允许人工审核");
        }
        AuditRecord record = new AuditRecord();
        record.setTaskId(taskId);
        record.setAuditType("MANUAL");
        record.setResult(approved ? 1 : 2);
        record.setReason(reason);
        record.setAuditorId(auditorId);
        recordMapper.insert(record);

        task.setStatus(approved ? 2 : 3);
        taskMapper.updateById(task);
        // 通知 property-service
        propertyClient.notifyAuditResult(task.getPropertyId(), approved);
    }
}
```

### 5. 回调 property-service 的客户端

java

```
@Component
@RequiredArgsConstructor
public class PropertyServiceClient {
    private final RestTemplate restTemplate;

    @Value("${property.internal.token:internal-token-xxx}")
    private String internalToken;

    public void notifyAuditResult(Long propertyId, boolean passed) {
        String url = "http://property-service/api/properties/" + propertyId + "/audit-result";
        AuditResultRequest request = new AuditResultRequest();
        request.setStatus(passed ? 1 : 3); // 1通过 3驳回
        request.setMessage(passed ? "审核通过" : "内容违规，审核驳回");

        HttpHeaders headers = new HttpHeaders();
        headers.set("X-Internal-Token", internalToken);
        headers.setContentType(MediaType.APPLICATION_JSON);

        HttpEntity<AuditResultRequest> entity = new HttpEntity<>(request, headers);
        restTemplate.exchange(url, HttpMethod.PUT, entity, Void.class);
    }
}
```



**AuditResultRequest**：

java

```
@Data
public class AuditResultRequest {

### 6. property-service 需提供的内部接口

java

```
@PutMapping("/api/properties/{id}/audit-result")
public Result<Void> updateAuditResult(@PathVariable Long id,
                                      @RequestBody AuditResultRequest request,
                                      @RequestHeader("X-Internal-Token") String internalToken) {
    // 校验内部 Token
    if (!"internal-token-xxx".equals(internalToken)) {
        return Result.error(403, "禁止访问");
    }
    propertyService.updateAuditStatus(id, request.getStatus());
    return Result.success();
}
```
