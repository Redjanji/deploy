**目标**：验证 analytics-service 宕机时，主业务流程不受影响。

**操作**：
1. 停止 analytics-service
2. 执行创建房源操作，确认 property-service 正常响应（StatsEventPublisher 的 try-catch 应捕获 MQ 连接异常并记录日志，但不阻断业务）
3. 检查 property-service 日志，确认有 "Failed to publish stats event" 错误日志但无异常抛出
4. 重新启动 analytics-service，确认服务恢复正常

## 假设与决策

1. **JWT_SECRET 值**：使用文档中的示例值 `x+6T9pB3rL8wQ1zCvN5mY2aD4fG7hJ0kL3oP6iR8uT4=`，gateway 和 auth 必须一致。
2. **APP_SECRET**：验证阶段需要获取 Token 调用受保护接口。若不设置 APP_SECRET，则通过 /auth/login 获取用户 JWT 直接调用内部服务（绕过网关 Token 机制）或设置临时 APP_SECRET。
3. **WSL IP**：使用 172.24.35.23（配置文件中的默认值）。
4. **验证脚本**：使用 PowerShell 脚本通过 [System.Net.Http.HttpClient] 调用 API（根据项目经验，Invoke-RestMethod 在 4xx/5xx 时无法可靠读取响应体）。
