# 新服务适配对接指南

## 一、服务架构概述

本项目采用微服务架构，基于 Spring Boot 3.2.x，包含以下核心组件：

| 组件 | 技术选型 | 用途 |
|------|---------|------|
| 服务注册/配置 | Nacos 2.2.x | 服务发现、配置管理 |
| API 网关 | Spring Cloud Gateway | 统一入口、路由、限流 |
| 认证授权 | Spring Security + JWT | 用户认证、Token 管理 |
| 数据库 | MySQL 8.x + MyBatis-Plus | 数据持久化 |
| 缓存 | Redis 7.x | 缓存、会话管理 |
| 消息队列 | RabbitMQ 3.12.x | 异步通信、事件驱动 |
| 对象存储 | MinIO | 图片/文件存储 |
| 全文检索 | Elasticsearch 8.x | 搜索功能 |

### 服务清单

| 服务名 | 端口 | 职责 |
|--------|------|------|
| gateway-service | 8080 | API 网关 |
| auth-service | 8081 | 用户认证 |
| dict-service | 8082 | 字典数据 |
| image-service | 8083 | 图片服务 |

## 四、认证授权机制

### 4.1 请求认证流程

```
客户端请求 → Gateway → HMAC签名验证 → JWT验证 → 转发到目标服务
```

### 4.2 应用级认证（HMAC）

所有请求需携带以下 Header：

| Header | 说明 | 必填 |
|--------|------|------|
| X-App-Id | 应用 ID | 是 |
| X-Timestamp | 时间戳（秒） | 是 |
| X-Nonce | 随机字符串 | 是 |
| X-Sign | HMAC-SHA256 签名 | 是 |

签名算法：
```
sign = HMAC-SHA256(appId + ":" + timestamp + ":" + nonce, appSecret)
```

### 4.3 用户级认证（JWT）

用户登录后获取 Token，后续请求携带：

```
Authorization: Bearer <token>
```

Token 解析后会注入以下 Header：

| Header | 说明 |
|--------|------|
| X-User-Id | 用户 ID |
| X-Owner-Id | 所有者 ID |

### 4.4 服务内认证

在 Controller 方法上使用 `@RequestHeader` 获取认证信息：

```java
@GetMapping("/items")
public Result<List<ItemVO>> getItems(
        @RequestHeader(value = "X-App-Id", required = false) String appId,
        @RequestHeader(value = "X-User-Id", required = false) Long userId) {
    // ...
}
```

## 五、API 设计规范

### 5.1 响应格式

统一使用 `Result<T>` 封装响应：

```java
public class Result<T> {
    private int code;
    private String message;
    private T data;
    
    public static <T> Result<T> success(T data) { /* ... */ }
    public static <T> Result<T> error(int code, String message) { /* ... */ }
}
```

响应示例：

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

### 5.2 错误码规范

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权 |
| 403 | 禁止访问 |
| 404 | 资源不存在 |
| 500 | 系统错误 |
| 10001 | 业务异常（自定义） |
| 10002 | 参数校验失败 |

### 5.3 路由规范

通过 Gateway 统一路由，格式为：

```
http://gateway:8080/{service-name}/{api-path}
```

示例：
- GET `/new-service/items` → 转发到 new-service
- POST `/new-service/items` → 转发到 new-service

### 5.4 Controller 示例

```java
package com.xss.newservice.controller;

import com.xss.newservice.common.Result;
import com.xss.newservice.dto.ItemCreateRequest;
import com.xss.newservice.service.ItemService;
import com.xss.newservice.vo.ItemVO;
import lombok.RequiredArgsConstructor;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@RestController
@RequestMapping("/items")
@RequiredArgsConstructor
public class ItemController {

    private final ItemService itemService;

    @GetMapping
    public Result<List<ItemVO>> list(
            @RequestParam(defaultValue = "1") Integer pageNum,
            @RequestParam(defaultValue = "10") Integer pageSize) {
        return Result.success(itemService.list(pageNum, pageSize));
    }

    @GetMapping("/{id}")
    public Result<ItemVO> getById(@PathVariable Long id) {
        return Result.success(itemService.getById(id));
    }

    @PostMapping
    public Result<ItemVO> create(@RequestBody ItemCreateRequest request) {
        return Result.success(itemService.create(request));
    }

    @PutMapping("/{id}")
    public Result<ItemVO> update(@PathVariable Long id, @RequestBody ItemCreateRequest request) {
        return Result.success(itemService.update(id, request));
    }

    @DeleteMapping("/{id}")
    public Result<Void> delete(@PathVariable Long id) {
        itemService.delete(id);
        return Result.success(null);
    }
}
```

## 六、异常处理规范

### 6.1 BusinessException

业务异常统一使用 `BusinessException`：

```java
package com.xss.newservice.common;

public class BusinessException extends RuntimeException {
    private int code;

    public BusinessException(String message) {
        super(message);
        this.code = 10001;
    }

    public BusinessException(int code, String message) {
        super(message);
        this.code = code;
    }

    public int getCode() {
        return code;
    }
}
```

### 6.2 GlobalExceptionHandler

全局异常处理器：

```java
package com.xss.newservice.config;

import com.xss.newservice.common.BusinessException;
import com.xss.newservice.common.Result;
import org.springframework.http.HttpStatus;
import org.springframework.validation.FieldError;
import org.springframework.web.bind.MethodArgumentNotValidException;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.ResponseStatus;
import org.springframework.web.bind.annotation.RestControllerAdvice;

import java.util.stream.Collectors;

@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(BusinessException.class)
    @ResponseStatus(HttpStatus.OK)
    public Result<Void> handleBusinessException(BusinessException e) {
        return Result.error(e.getCode(), e.getMessage());
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    @ResponseStatus(HttpStatus.OK)
    public Result<Void> handleValidationException(MethodArgumentNotValidException e) {
        String message = e.getBindingResult().getFieldErrors().stream()
                .map(FieldError::getDefaultMessage)
                .collect(Collectors.joining(", "));
        return Result.error(10002, message);
    }

    @ExceptionHandler(Exception.class)
    @ResponseStatus(HttpStatus.OK)
    public Result<Void> handleException(Exception e) {
        return Result.error(500, "系统错误");
    }
}
```

### 6.3 使用示例

```java

    ItemEntity item = itemMapper.selectById(id);
    if (item == null) {
        throw new BusinessException("资源不存在");
```

## 七、服务间调用

### 7.1 使用 RestTemplate

配置 `RestTemplateConfig`：

```java
@Configuration
public class RestTemplateConfig {

    @Bean
    @LoadBalanced
    public RestTemplate restTemplate() {
        return new RestTemplate();
    }
}
```

调用其他服务：

```java
@Service
@RequiredArgsConstructor
public class ItemService {

    private final RestTemplate restTemplate;

    public UserInfoDTO getUserInfo(Long userId) {
        String url = "http://auth-service/auth/userinfo/{userId}";
        ResponseEntity<Result<UserInfoDTO>> response = 
                restTemplate.getForEntity(url, new ParameterizedTypeReference<Result<UserInfoDTO>>() {}, userId);
        return response.getBody().getData();
    }
}
```

## 八、MQ 消息机制

### 8.1 发布消息

配置 RabbitMQ：

```java
@Configuration
public class RabbitMQConfig {

    @Bean
    public Queue eventQueue() {
        return new Queue("new-service.event", true);
    }
}
```

发布消息：

```java
@Service
@RequiredArgsConstructor
public class EventPublisher {

    private final RabbitTemplate rabbitTemplate;

    public void publishEvent(String eventType, Long targetId) {
        Map<String, Object> event = new HashMap<>();
        event.put("type", eventType);
        event.put("targetId", targetId);
        event.put("timestamp", System.currentTimeMillis());
        rabbitTemplate.convertAndSend("new-service.event", event);
    }
}
```

### 8.2 消费消息

```java
@Component
public class EventConsumer {

    @RabbitListener(queues = "new-service.event")
    public void handleEvent(Map<String, Object> event) {
        String eventType = (String) event.get("type");
        Long targetId = ((Number) event.get("targetId")).longValue();
        
        switch (eventType) {
            case "ITEM_CREATE":
                handleItemCreate(targetId);
                break;
            case "ITEM_UPDATE":
                handleItemUpdate(targetId);
                break;
            default:
                // 未知事件类型
        }
    }
}
```

## 十二、常见问题

### Q1：如何获取用户认证信息？

通过 `@RequestHeader` 获取：

```java
@GetMapping("/items")
public Result<List<ItemVO>> list(
        @RequestHeader(value = "X-User-Id", required = false) Long userId) {
    // userId 为 null 表示匿名访问
}
```

### Q2：如何调用其他服务？

使用 `RestTemplate` + 服务名：

```java
String url = "http://auth-service/auth/userinfo/{userId}";
```

### Q3：异常应该抛什么类型？

业务逻辑异常使用 `BusinessException`，参数校验异常由 Spring 自动处理。

### Q4：测试时需要启动所有依赖服务吗？

单元测试使用 Mock，不需要启动依赖服务；集成测试需要启动相关服务。

### Q5：如何配置 Gateway 路由？

在 gateway-service 的 `application.yaml` 中添加：

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: new-service
          uri: lb://new-service
          predicates:
            - Path=/new-service/**
          filters:
            - StripPrefix=1
```
